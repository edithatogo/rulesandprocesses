4. How would you model rules/process coupling without making rules non-reusable?
5. Is "superset" the right framing, or should the project be framed as a crosswalk/profile/conformance layer?
6. How should a public concept registry be governed and versioned?
7. What would an Axiom-first adoption path look like?
8. How should AI-assisted migration be constrained by deterministic validation?
9. What would you drop from the current conceived repo?
10. What would you add to make it credible to policy economists and microsimulation users?
11. What would you add to make it credible to administrative lawyers and implementation teams?
12. What are the highest-value tests in the first 90 days?
